# 日本語コミュニケーション設定

## 🇯🇵 言語設定

### 基本方針
- **すべてのやりとりは日本語で行う**
- 技術用語は適切な日本語訳を使用し、必要に応じて英語を併記
- コードコメントは日本語で記述
- エラーメッセージは日本語で提供

### コミュニケーションスタイル

#### 敬語・丁寧語の使用
- ユーザーに対しては丁寧語を使用
- 「です・ます調」で統一
- 専門的な内容も分かりやすく説明

#### 技術用語の日本語化
- **Clean Architecture** → クリーンアーキテクチャ
- **Domain-Driven Design** → ドメイン駆動設計
- **Test Coverage** → テストカバレッジ
- **Code Review** → コードレビュー
- **Refactoring** → リファクタリング

## 🎯 専門用語集

### アーキテクチャ関連
- **Entity** → エンティティ（ビジネスオブジェクト）

- **Repository** → リポジトリ（データアクセス層）
- **Adapter** → アダプター（外部システム接続）
- **Dependency Injection** → 依存性注入

- **Linting** → 静的解析（コード品質チェック）

### 品質管理関連
- **Technical Debt** → 技術的負債

- **Quality Gate** → 品質ゲート（品質基準チェック）

  xpath: '//*[@id="invalid"]',

});

// エラーメッセージ（messages.json）
{
  "E_XPATH_0001_USER": {

  },

    "message": "XPathセレクターを確認し、対象要素が存在することを確認してください",

  }

```

## 🌐 多言語対応

### 対応言語
- **プライマリ**: 日本語
- **セカンダリ**: 英語（技術用語、コード内）

### リソースファイル構造
```
public/_locales/
├── ja/
│   └── messages.json    # 日本語メッセージ
└── en/
    └── messages.json    # 英語メッセージ（フォールバック）
```

### メッセージキー命名規則
```json
{
  "feature_action_element_ja": {
    "message": "日本語メッセージ",
    "description": "このメッセージの用途説明（日本語）"
  }
}
```
